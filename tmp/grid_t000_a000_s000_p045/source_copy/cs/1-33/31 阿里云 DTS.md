## 概要
Data Transmission Service，阿里云的数据库数据传输服务

支持==同构和异构==的数据源之间的==数据迁移和实时同步==
例如
1. mysql->mysql
2. mysql->kafka
3. [[30 阿里云 RDS]]->[[30.1 阿里云 DRDS(PolarDB-X)]]
4. mysql->ES
5. 

场景：
1. 不停机迁移数据库
2. [[Pasted image 20210330122346.png|数据库异地灾备]]
3. 利用双向同步实现异地多活，实现全局数据一致性 [[Pasted image 20210330122530.png|异地多活]]
4. [[Pasted image 20210330122706.png|加速全球访问]]
5. [[Pasted image 20210330130821.png|实时数据分析]]
6. 将数据库映射到缓存系统[[Pasted image 20210330130858.png|缓存]]
7. [[Pasted image 20210330135928.png|业务解耦合]]
## 迁移原理
全量迁移+增量迁移，类似于redis的同步过程，包含一个复制缓冲区
![[Pasted image 20210330121337.png|500]]

## 实时同步
增量同步的过程是异步的，源数据库的binlog会被先落成dts本地的日志，再由单独的日志消费模块消费并写入目标数据库
![[Pasted image 20210330121625.png|500]]

## 数据订阅
和实时同步类似，但由用户使用dts sdk自行决定binlog的使用方式
支持使用多个消费者实现高可用，但只有一个消费者可以获取到数据
