## 概述
一种轻量级，基于 DDD 概念的系统分析方法

## 核心概念
事件风暴的核心流程就是
user执行了action，产生了event

### event
事件风暴的核心概念
1. 代表一个业务行为
2. 描述为『宾语+动词过去式』
3. 例如『订单被提交』，『商品被发出』

### command
1. 命令是产生事件的动作，执行后会产生相应的对象
2. 例如『支付订单』，『取消订单』

## user/actor
用户，发出命令的主体

## policy
业务规则

## read model 和 screen layerout
layerout表示对用户的数据展示，read model是layerout对应的数据的的表示

## system
外部系统，event不一定由command产生，也可能是外部系统产生的

## question和assumption
讨论中的分期，先用这样的概念卡片pending住

## aggregate
这里和ddd的aggregate应该不是一个含义，指的是一个包含user，command，event的完整场景，「用户在 XX 聚合对象上执行了 YY 命令，生成了 ZZ 事件」

例如：「顾客在购物车对象上执行了结账命令，生成了购物车结算事件」
