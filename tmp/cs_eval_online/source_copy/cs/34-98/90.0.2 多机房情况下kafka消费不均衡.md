# consumer和partition数量不匹配
- consumer实例数不能被订阅的某一个集群的topic的分区数整除
- consumer实例数大于分区数

# 两个kafka集群独立reblance（快手az环境下）
如果topic为多集群，则要保证每个集群的topic分区数都能整除consumer实例数
![[Pasted image 20240910114450.png|400]]
[[172 CPU性能问题 耗时突增分析（java，容器环境下）#AZ间的CPU分层]]

一个实际的例子：
- 两个机房各有20个partition，共40个
- 最初有每个机房15个consumer，发现部分机器消费了两个partition

这里是因为partition数量>consumer数量，符合预期
但是把consumer调整到40之后，仍然存在一个consumer消费多个partition的情况，不符合预期

这里涉及consumer的两种消费模式，默认都是采用的current region模式，也就是一个consumer会消费多个az的数据
![[Pasted image 20240910152252.png]]
这时就导致了上面讲的每个机器的topic会出现独立reblance的情况，导致consumer消费的不平衡
![[Pasted image 20240910152431.png]]

解决办法就是让所有az的consumer的数量总和=单个az的topic的partition数量的倍数
