## 内存分配的基础知识
- 操作系统基于page来管理内存，linux下一个page为4k
- 每个进程使用一个虚拟的地址，会通过mmu来转换到物理的地址
- 物理地址和虚拟地址之间的映射都是以page作为单位的，offset两边一致
- page_id + offset用来表示一个具体的地址

### linux的进程内存管理
单个linux进程的内存使用
![[Pasted image 20211109151320.png]]
1. code：最底部，编译好的可执行机器码
2. data：初始化过的全局变量
3. bss：未初始化的全局变量
4. head：堆，从低地址到高地址增长
5. mapping area：mmap调用的相关区域
6. stack：栈，从高地址向低地址增长

#### heap
malloc分配的内存主要在head中（大块内存可能通过mmap申请）
![[Pasted image 20211109151952.png|400]]
break指针维护以映射区域和微映射区域的边界，mapped region区域已经实际映射到物理页，对unmapped region区域的访问会报错

break指针需要通过brk和sbrk系统调用操作
```c
int brk(void *addr) //直接设置一个新的break的位置
void *sbrk(intptr_t increment) //设置一个步进值，返回最新的break地址
```
操作系统会在break设置后自动进行page的4k对齐，所以break之后可能有一小段空间是可以被访问的，但是非常不安全

进程的可用内存大小是由rlimit决定的，包含硬限和软限
