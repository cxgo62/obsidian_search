## 概述
对[[126 embedding]]的改进，解决对网络化数据的支持

最终生成的向量包含
1. 图的结构信息
2. 附近节点的局部相似性信息

## 几种构建方式

### deep walk
在dag中随机游走，产生一些随机的路径序列，并使用基础的embedding方式训练


### node2vec
通过调整随机游走权重的方法使 Graph Embedding 的结果更倾向于体现网络的同质性(homophily ) 或结构性( structural equivalence ) 。

表达结构性：
1. 距离相近节点的 Embedding 应尽量近似[](marginnote3app://note/35E05848-68F8-4F72-B598-ED7A6ECBFEA6)
2. 让游走过程更倾向于BFS

表达同质性：
1. 结构上相似的节点的 Embedding 应尽量近似
2. 让游走过程更倾向于DFS

通过控制跳转的概率来选择结构性或同质性的倾向

同质性相同的物品很可能是同品类、同属性,或者经常被一同购买的商品, 而结构性相同的物品则是各品类的爆款、 各品类的最佳凑单商品等拥有类似趋势或者结构性属性的商品

embedding层的参数多，收敛慢，而且变动的可能性相对低，所以不适合直接作为一层加在深度学习模型中进行训练
一般会使用训练好的embedding来做深度学习数据的预处理
