## 概论
两者都是通过随机实现的一致性hash算法

一致性hash的指标
1. 桶数量变化后的映射关系变化率
2. 每个桶分布的均匀性
## jumping hash
比较简单的一致性算法，如何把key分布到n个桶中
```cpp
int ch(int key, int n)
{
    random.seed(key);
    int b = 0;
    int f = 0;
    while (f < n)
    {
        b = f;
        r = random.next();
        f = floor((b+1)/r)
    }
    return b;
}
```

public class CrushSelector {  
    final private HashMap<String, Integer> candidates = new HashMap();  
  
 public void addCandidate(String candidate, int weight) {  
        candidates.put(candidate, weight);  
 }  
  
    public void addCandidatesBatch(HashMap<String, Integer> candidates_) {  
        candidates.putAll(candidates_);  
 }  
  
    public String getWinner(String userKey, int sequence) {  
        String minKey = "";  
 double minScore = 10000000;  
  
 for (Map.Entry<String, Integer> entry : candidates.entrySet()) {  
            String candidate = entry.getKey();  
 double score = (1- Math.log(getRandomNumer(candidate, userKey, sequence))/entry.getValue());  
 if (score < minScore) {  
                minScore = score;  
 minKey = entry.getKey();  
 }  
        }  
  
        return minKey;  
 }  
  
    private double getRandomNumer(String candidate, String userKey, int sequence) {  
        String seedKey = candidate + userKey + sequence;  
  
 Random random = new Random(seedKey.hashCode());  
 return random.nextDouble();  
 }  
}

### 原理
有三个输入

1. candidate 可能的目标机器
2. userKey 用户的key
3. sequence 用来选择备份机器的序列号

这三个参数作为输入，可以通过hash获得一个随机的值

> 这个hash函数好像比较特别，随便写一个效果不好，这里将这三个参数作为seed的random函数来代替，效果上差别不大，性能可能差一点

同一个（userkey，sequence）和所有的candidate（n个）组合，将得到n个随机值，可以选取最小的这个值对应的candidate作为获胜者

所以crush这种方式也被叫做抽签算法，其实就是把多个组合（userkey、candidate、sequence）拿去抽签，看谁小就选谁

### 如何保证一致性hash
因为这个hash函数是固定的，所以单个（userkey、candidate、sequence）得到的分是一个固定值，当candidate的集合不变的时候，这个最后的winner是一定的
当新的candidate加入时，因为必须刚好（userkey、new_candidate、sequence）是全局最低得分才会替代原有的winner，所以从概率上讲，大部的winner是不便的，也就实现了一致性hash


### 如何设置candidate的权重
因为是最小的得分获胜，所以可以将各个组合的得分除以各个candidate的权重，这样权重越大的candidate可能成为winner的概率就越高

score = hash(userkey,new_candidate,sequence)/weight

但实际这样的效果只能保证weight大的概率高，不能保证结果一定是按照weight分布的
所以必须再加一步额外的处理

score = （1-log(hash(userkey,new_candidate,sequence)))/weight
