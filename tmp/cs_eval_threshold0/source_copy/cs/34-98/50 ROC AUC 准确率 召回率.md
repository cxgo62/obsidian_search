## 混淆矩阵
二分类问题中可能出现的四种结果
![[Pasted image 20210315160026.png|500]]
定义：
positive VS negative
在二分类问题中：
- positive：预测值为1
- negative： 预测值为0

True VS False
- True 预测正确
- False 预测错误

真阳率：
- 所有真实为1的样本中，预测正确的概率
- TPRate = TP/(TP+FN)
- TPRate == recall 召回率
- 用来判断是否找的全，有 1 - TPRate的概率模型会漏判

伪阳率：所有真实为0的样本中，预测错误的概率
FPRate = FP/(FP+TN)

真阳率可以理解为正确预测率，伪阳率可以理解为误杀率

另外的几个概念:
1. 准确率
	- 预测对的占整个的比例
	- accuracy = （TP + TN）/ (P + N)

2. 精确度
	- 所有预测为1的概率的样本，预测正确的概率
	- 评价是否找的对
	- precision = TP/（TP+FP）

3.召回率：recall == TPRate

### 一个召回率和精确度的例子
例子：一个攻击检测模型，真实的攻击为1，正常请求为0
- 当一个模型的召回率为95%时，代表有5%的攻击会被漏掉，被判断为无害
- 精确度为90%，代表在所有被判断为攻击的请求中，有10%的是正常请求，也就是误杀率是10%
- FPRate和1-precision都可以表示误杀率，但是两个的分母不同，前者是实际为0的样本数，后者是实际被判断为positive的数量


## AUC ROC

**ROC（receiver operating characteristic curve）**接收者操作特征曲线
横坐标为伪阳率FPRate，纵坐标为真阳率TPRate的曲线
ROC上的一个点可以认为是在一个threshold下的（FPRate，TPRate）值
ROC表现的是模型对所有threshold的表现的总和


**AUC**（Area Under Curve）被定义为ROC曲线下的面积

### 为什么要使用AUC
在二分类场景下，当使用accuracy来预测模型时，必须要先设定一个threshold，threshold的设定很大程度的影响了模型的评估，AUC可以避免使用threshold

auc的数学解释（Mann–Whitney U statistic）：
1. 从所有1样本中随机选取一个样本
2. 从所有0样本中随机选取一个样本
3. 使用分类器对两个样本进行预测
4. 1样本被预测为1的概率为p1
5. 0样本被预测为1的概率为p0
6. ==p1 > p0 的概率，即正确率大于误杀率的概率即为AUC==，或者表诉为：任取一对例和负例，正例得分大于负例得分的概率

所以当使用随机预测器时，p1 > p0的概率为0.5 符合之前的定义

AUC的另外一个优势，对==正负样本的均衡不敏感==
例如在反欺诈场景，设欺诈类样本为正例，正例占比很少（假设0.1%），如果使用准确率评估，把所有的样本预测为负例，便可以获得**99.9%的准确率**。

但是如果使用AUC，把所有样本预测为负例，TPRate和FPRate同时为0（没有Positive），与(0,0) (1,1)连接，得出**AUC仅为0.5**，成功规避了样本不均匀带来的问题。
